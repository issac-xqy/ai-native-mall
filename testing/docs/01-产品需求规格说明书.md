## 1. 产品概述

### 1.1 产品定位
AI-Native 智能商城是一站式 AI 驱动电商平台，将大模型能力深度嵌入商品运营、智能客服、语义搜索等核心场景。

### 1.2 目标用户
| 角色 | 描述 |
|------|------|
| C端消费者 | 浏览/搜索/购买商品，使用AI客服咨询 |
| 运营/管理员 | 管理商品/分类/订单，使用AI工具优化运营效率 |

### 1.3 产品边界
- **包含**: 商品浏览、购物车、下单支付、AI客服、AI商品运营、语义搜索、钱包、后台管理
- **不包含**: 第三方支付对接(仅钱包余额支付)、物流系统对接、IM即时通讯

## 2. 功能需求

### 2.1 用户认证模块 (P0)

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|----------|
| F-USER-001 | 用户注册 | 用户名+手机号+密码注册 | 注册成功返回Token, BCrypt加密存储 |
| F-USER-002 | 用户登录 | 用户名/手机号+密码登录 | 返回JWT Token, 24小时有效 |
| F-USER-003 | 获取用户信息 | 根据Token获取当前用户信息 | 返回昵称/手机/邮箱/头像 |
| F-USER-004 | 更新用户信息 | 修改昵称/邮箱/头像 | 修改成功, 敏感信息脱敏 |

### 2.2 商品展示模块 (P0)

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|----------|
| F-PROD-001 | 商品列表 | 分页展示, 支持分类/关键词筛选, 价格/销量/时间排序 | 默认按创建时间降序, 只显示已上架且有库存商品 |
| F-PROD-002 | 商品详情 | 展示名称/价格/库存/描述/规格参数/AI评价 | 浏览量+1 |
| F-PROD-003 | 分类树 | 多级分类展示 | 支持父子层级 |
| F-PROD-004 | 销量Top10 | 首页推荐热销商品 | 按sales降序, 已上架有库存 |
| F-PROD-005 | 好评Top10 | 首页推荐好评商品 | 按评论平均评分降序 |

### 2.3 购物车与订单模块 (P0)

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|----------|
| F-CART-001 | 加入购物车 | 选择商品和数量加入购物车 | 同一用户同一商品合并数量 |
| F-CART-002 | 修改数量/删除 | 增加/减少/删除购物车商品 | 实时更新总价 |
| F-ORDER-001 | 创建订单 | 从购物车选择商品, 填写地址后下单 | 扣库存, 生成订单号 |
| F-ORDER-002 | 钱包支付 | 使用钱包余额支付订单 | 余额足够则扣款成功, 订单状态变为已支付 |
| F-ORDER-003 | 订单列表 | 查看个人所有订单 | 按时间降序, 显示状态 |
| F-ORDER-004 | 确认收货 | 用户确认收到商品 | 订单状态变为已完成 |
| F-ORDER-005 | 取消订单 | 待支付状态可取消 | 恢复库存, 如已支付则退款 |
| F-ORDER-006 | 申请退款 | 已支付/已发货可退款 | 退款到钱包余额 |

**订单状态机:**
```
待支付(0) ──支付──→ 已支付(1) ──发货──→ 已发货(2) ──确认──→ 已完成(3)
    │                  │                    │
    └──取消──→ 已取消(4)  └──退款──→ 已取消(4)  └──退款──→ 已取消(4)
```

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 目标 |
|------|------|
| 商品列表查询 QPS | ≥500 |
| 商品列表 P99 延迟 | <200ms |
| 订单创建 TPS | ≥100 |
| AI客服流式首字延迟 | <2s |
| 页面加载时间(FCP) | <1.5s |

### 3.2 安全要求

| 要求 | 实现 |
|------|------|
| 密码加密 | BCrypt |
| 接口鉴权 | JWT (24h有效) |
| SQL注入防护 | MyBatis-Plus参数化 |
| XSS防护 | Element Plus自动转义 |
| 文件上传安全 | 类型白名单 + 大小限制 |
| AI API Key保护 | 环境变量 + 不返回前端 |
| 敏感词过滤 | AI对话输入检测 |

### 3.3 兼容性

| 项目 | 要求 |
|------|------|
| JDK | 21 LTS |
| 数据库 | MySQL 8.0+ |
| 浏览器 | Chrome/Firefox/Edge 最新2版 |
| 移动端 | 用户端适配 375px 宽屏 |

### 3.4 可用性

| 指标 | 目标 |
|------|------|
| 系统可用率 | 99.9% |
| AI服务可用率 | 99.5% (含降级) |
| 数据持久性 | 99.99% |
