## 2. AI 模型配置

### 2.1 模型清单

| 模型 | 供应商 | 用途 | API URL | 配置Key |
|------|--------|------|---------|---------|
| qwen-max | 通义千问 | 主对话/内容生成/意图识别 | `dashscope.aliyuncs.com/compatible-mode/v1` | `AI_API_KEY` |
| deepseek-chat | DeepSeek | 备用对话模型 | `api.deepseek.com/v1` | `DEEPSEEK_API_KEY` |
| text-embedding-v4 | 通义千问 | 文本向量化 | `dashscope.aliyuncs.com/compatible-mode/v1` | `AI_API_KEY` |
| all-MiniLM-L6-v2 | ONNX 本地 | 离线向量化备用 | 本地加载 | 无需API Key |

### 2.2 模型参数

```yaml
# 对话模型
model: qwen-max
temperature: 0.7        # 创造性控制 (0=确定, 1=随机)
max-tokens: 2000        # 单次最大输出
top-p: 0.8              # 核采样

# 向量模型
embedding-model: text-embedding-v4
embedding-dimension: 1536
use-external-embedding: true    # false=使用本地ONNX模型
auto-clear-vector-store: true  # 维度不匹配时自动重建
```

## 3. AI 能力详解

### 3.1 智能客服 (SmartCustomerService)

**核心流程: RAG 检索增强生成**

```
用户问题
  │
  ▼
敏感词检测 (sensitive-word库)
  │
  ├── 命中 → 返回拒绝提示
  │
  └── 通过
       │
       ▼
  意图识别 (IntentRecognitionService)
       │
       ▼
  向量检索 (ContentRetriever, Top-5, minScore=0.5~0.7)
       │
       ▼
  构建 Prompt (系统提示词 + 检索结果 + 对话历史 + 用户问题)
       │
       ▼
  qwen-max 生成回答
       │
       ▼
  SSE 流式返回 (逐token推送)
```

**两种接口:**

| 模式 | 接口 | 适用场景 |
|------|------|----------|
| 同步 | `POST /api/ai/customer-service/ask` | 简单问答, 快速响应 |
| 流式 | `GET /api/ai/customer-service/stream` | 长回答, 打字机效果 |

**RAG 参数:**

| 参数 | 值 | 说明 |
|------|-----|------|
| Top-K | 5 | 检索最相似5条 |
| Min Score | 0.5~0.7 | 相似度阈值(可调) |
| 相似度度量 | COSINE | 余弦相似度 |

**系统提示词策略:**
- 限定角色: "你是AI-Native智能商城的客服助手"
- 知识边界: "只回答基于知识库的商品和商城政策问题"
- 拒绝策略: "不知道的问题诚实说不知道"

### 3.2 商品运营 (ProductOperationService)

| 能力 | 接口 | Prompt策略 | 输出 |
|------|------|------------|------|
| SEO标题 | `POST /api/ai/product/seo-title` | 提取商品名+核心卖点+分类词, 30字以内 | SEO标题字符串 |
| 商品描述 | `POST /api/ai/product/description` | 基于specs+description生成营销文案, 突出差异化卖点 | 200-500字文案 |
| 评论分析 | `POST /api/ai/comment/analyze` | 判断正/负/中性, 提取3-5个关键词标签, 总结主要观点 | sentiment + tags + summary |
| 口碑报告 | `POST /api/ai/product/reputation-report` | 批量分析N条评论, 统计好评率, 提取高频词, 给出改进建议 | JSON结构报告 |
| 关联推荐 | (AIRecommendController) | 基于商品特征相似度+用户行为, 生成推荐理由 | 商品列表+理由 |

### 3.3 语义搜索 (ProductSemanticSearchService)

**混合搜索策略:**

```
用户查询 "性价比高的手机"
  │
  ├── 向量搜索路: Embedding(查询) → Redis向量相似度 → Top-20
  │
  ├── 全文搜索路: 关键词分词 → Elasticsearch → Top-20
  │
  └── 融合排序: 加权合并(向量分×0.6 + 关键词分×0.4) → Top-N
```

### 3.4 意图识别 (IntentRecognitionService)

**10种意图分类:**

| 意图 | 英文标识 | 典型问法 |
|------|----------|----------|
| 商品咨询 | PRODUCT_INQUIRY | "这个手机有什么功能" |
| 价格咨询 | PRICE_INQUIRY | "iPhone多少钱" |
| 砍价 | BARGAINING | "能不能便宜点" |
| 订单状态 | ORDER_STATUS | "我的订单到哪了" |
| 退货政策 | RETURN_POLICY | "怎么退货" |
| 物流查询 | LOGISTICS_TRACKING | "快递单号是多少" |
| 发票 | INVOICE_REQUEST | "能开发票吗" |
| 优惠券 | COUPON_INQUIRY | "有优惠吗" |
| 闲聊 | CHITCHAT | "今天天气真好" |
| 未知 | UNKNOWN | 无法归类 |

### 3.5 知识库管理 (KnowledgeBaseService)

```
用户上传文档 (PDF/DOCX/TXT)
  │
  ▼
DocumentParserService 解析
  ├─ PDF  → PDFBox 提取文本
  ├─ DOCX → POI 提取文本
  └─ TXT  → 直接读取
  │
  ▼
文本分块 (splitIntoChunks)
  ├─ 按段落优先 (保留语义完整性)
  └─ 最大块大小: 1000字符
  │
  ▼
向量化 (EmbeddingModel.embed)
  │
  ▼
存储到 Redis Vector Store
  │
  ▼
更新状态: status = 1 (已向量化)
```

## 4. AI 安全与降级

### 4.1 安全措施

| 措施 | 实现 |
|------|------|
| API Key保护 | 环境变量, 不写代码, 不返前端 |
| 输入过滤 | sensitive-word 库 (0.21.0) |
| 输出审查 | 敏感词检测 |
| 速率限制 | Redis滑动窗口 + Sentinel QPS=20 |
| Token上限 | 单次对话最大2000 tokens |

### 4.2 降级策略

```
AI API 调用
  │
  ├── 正常响应 → 返回结果
  │
  ├── 超时 (30s) → 返回 "服务繁忙, 请稍后重试"
  │
  ├── API返回错误 → 返回 "AI助手暂时不可用, 请联系人工客服"
  │
  ├── 速率限制触发 → 返回 "咨询人数较多, 请稍后再试"
  │
  └── 网络不可达 → 回退到本地规则匹配 (FAQ关键词匹配)
```

### 4.3 AI 监控指标

| 指标 | 来源 | 告警阈值 |

| 知识库缺失率 | AiMonitorAspect | >30% 提示补充 |
| Token消耗 | AiMonitorAspect | 每日统计 |

## 5. Embedding 模型切换方案

### 5.1 切换流程

```
配置 use-external-embedding=true
  │
  ▼
使用 text-embedding-v4 (1536维, DashScope API)
  │
  ▼
VectorStoreDimensionChecker 启动时检查
  ├── 存量向量维度=1536 → 正常使用
  └── 存量向量维度≠1536 → auto-clear → 重建索引
```

### 5.2 本地 ONNX 模型 (离线备用)

```
配置 use-external-embedding=false
  │
  ▼
使用 all-MiniLM-L6-v2 (384维, ONNX Runtime)
  │
  ▼
优势: 离线可用, 零API费用
劣势: 维度更低, 语义精度不如云端模型
```
